# PRD：线下展 MMD/Vocaloid 工程寄存与匹配码回取模块

## 1. 背景
在线下展摊位场景中，用户会产生 MMD/Vocaloid 工程文件与导出成品（视频、音频、工程包）。现场直接传文件容易混淆、用户离开后无法回取。

目标是提供一个“上传即生成匹配码、后续凭码下载”的轻量流程。

## 2. 目标与指标

### 2.1 业务目标
- 用户无需注册，1~2 分钟内完成上传。
- 上传完成后获得唯一匹配码（6~8 位）。
- 用户可在有效期内凭匹配码二次下载。

### 2.2 指标（MVP）
- 上传成功率 > 99%
- 匹配码查询平均响应 < 1 秒（不含下载）
- 错误码输入时返回明确提示

## 3. 用户角色
- 普通用户：上传、回取下载
- 摊位管理员：查询记录、手动失效、辅助找回

## 4. 用户流程
1. 打开网站首页，进入上传页
2. 上传工程/导出文件（支持多文件）
3. 系统生成匹配码 + 到期时间 + 下载入口
4. 用户保存匹配码（复制、二维码、拍照）
5. 后续在“匹配码下载”入口输入匹配码并下载

## 5. 功能范围（MVP）

### 5.1 上传
- 支持多文件上传
- 记录文件元数据（名称、大小、类型、对象存储 key）
- 生成唯一 match code

### 5.2 回取下载
- 输入 match code 查询记录
- 展示文件列表并下载
- 支持过期提示与无效码提示

### 5.3 管理员基础能力
- 查询 upload record
- 手动失效/删除记录（后续 API）

## 6. 数据结构（核心）
- upload_record
  - id
  - booth_id
  - match_code
  - created_at / expires_at
  - status (active/expired/deleted)
  - download_count
  - metadata（昵称/联系方式后四位/备注）
- upload_file
  - id
  - record_id
  - file_name
  - size
  - mime_type
  - object_key
  - checksum

## 7. 安全与风控
- 匹配码使用高可读字符集，排除易混字符（0/O/1/I/L）
- 查询接口限流 + 防暴力穷举
- 过期自动清理
- 全站 HTTPS

## 8. 版本规划
- V1（本次）：
  - 核心领域模型
  - 匹配码生成
  - 上传记录创建与查询
  - 内存存储适配器（开发验证）
- V2：
  - DB 持久化（PostgreSQL/MySQL）
  - 对象存储适配器（S3/OSS）
  - 管理后台接口
- V3：
  - 二维码票据、下载日志、限流策略、病毒扫描

## 9. 验收标准
- 创建上传记录会返回唯一匹配码
- 根据匹配码可查询记录
- 过期记录查询时状态为 expired
- 下载后 download_count 可递增
